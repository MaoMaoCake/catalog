## About this Helm chart

This is a Kyverno Policy Reporter Docker Hardened Helm chart built from the upstream Policy Reporter Helm chart and
using a hardened configuration with Docker Hardened Images.

The following Docker Hardened Images are used in this Helm chart:

- `dhi/kyverno-policy-reporter`
- `dhi/kyverno-policy-reporter-ui`

To learn more about how to use this Helm chart you can visit the upstream documentation:
[https://kyverno.github.io/policy-reporter/](https://kyverno.github.io/policy-reporter/)

## About Kyverno Policy Reporter

Kyverno Policy Reporter is a monitoring and observability tool that provides visibility into Kubernetes policy engine
results. It collects and processes PolicyReport and ClusterPolicyReport resources generated by policy engines like
Kyverno, Falco, or Trivy, and makes this data available through multiple channels for monitoring, alerting, and
analysis.

Policy Reporter can export metrics to Prometheus for tracking compliance trends over time, send notifications to various
targets including Slack, Discord, Loki, and Elasticsearch when policy violations occur, and provide an optional web
interface for browsing and analyzing policy reports. It supports multi-tenancy with namespace-scoped reporting and can
aggregate policy reports from multiple clusters, making it an essential tool for maintaining visibility into your
cluster's security and compliance posture.

Official documentation: https://kyverno.github.io/policy-reporter/

## About Docker Hardened Images

Docker Hardened Images are built to meet the highest security and compliance standards. They provide a trusted
foundation for containerized workloads by incorporating security best practices from the start.

### Why use Docker Hardened Images?

These images are published with near-zero known CVEs, include signed provenance, and come with a complete Software Bill
of Materials (SBOM) and VEX metadata. They're designed to secure your software supply chain while fitting seamlessly
into existing Docker workflows.
